However, for very large data sets greater than 2,000 we do have a search text output property that the modern combo box control exposes that we can leverage for serverside filtering of data meaning delegable queries. So with this our control can work with large data.

---

KEY POINTS:
- **Problem**: Default search only searches loaded items (max 2,000), not full dataset
- **Proof**: "Prepare" search misses item that exists beyond 2,000 row limit
- **Solution**: `SearchText` output property for server-side filtering
- Delegable queries work with large datasets
- SharePoint limitation: `Search()` not delegable
- SharePoint solution: Use `Filter()` + `StartsWith()`
- Formula pattern: `Filter(DataSource, StartsWith(Title, Self.SearchText))`
- Result: Server-side query returns matches from ALL 9,000+ rows
