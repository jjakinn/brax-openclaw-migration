KEY POINTS:
- Security audit: `openclaw security audit` + `openclaw security audit-fix`
- Critical: Web UI should NOT be exposed to internet
- Default: Web UI bound to 127.0.0.1 (localhost only)
- Access via SSH tunnel, not public IP
- Firewall hardening: Enable UFW, allow only SSH (port 22)
- Approval workflow: Agent asks before executing (can be disabled)
- Tool Profiles:
  - `coding` — Limited: files, terminal, no browsing/search
  - `full` — Agent sees ALL tools
- Check: `openclaw config get tools.profile`
- Change: `openclaw config set tools.profile full`
- Restart gateway after changes: `openclaw gateway restart`
