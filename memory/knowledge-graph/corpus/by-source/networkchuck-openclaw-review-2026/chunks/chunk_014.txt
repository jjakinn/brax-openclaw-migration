KEY POINTS:
- Execution Mode (`tools.exec.security`):
  - `full` — Agent can use any tool freely
  - `allowlist` — Only tools on approved list
  - `deny` — Blocked tools (distrust mode)
  - `ask` — Prompt before execution (paranoid mode)
  - `ask: on-miss` — Ask only if not on allowlist
- Red Lines (in agents.md):
  - **Red**: STOP, ask first (e.g., modify SSH config, exfiltrate data)
  - **Yellow**: Do it, but log it (e.g., firewall changes, Docker)
  - **Green/Always**: No restriction
- IMPORTANT: Red lines are PROMPTS, not deterministic enforcement
- Agent CAN disobey — there's no hard enforcement
- Security model: Trust + monitoring, not hard guarantees
