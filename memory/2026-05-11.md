## Fix 1: Body/Head Not Loading in Character Creator

**Bug:** Only hair rendered; body and head were invisible.

**Root Cause:** `Color.distance_to()` does NOT exist in Godot 4.x. Script error silently returned null, LPCLayer set `visible = false`.

**File:** `scripts/CharacterCreator.gd` → `_recolor_texture()`
- Broken: `if pixel.distance_to(src) < 0.1:`
- Fixed: `if pixel.is_equal_approx(src):`

**Note:** `Vector2.distance_to()` still works in Godot 4.x. Only `Color.distance_to()` was removed.

## Fix 5: Idle Duplicate Character — Two Characters Visible When Standing Still

**Bug:** When idle (not walking), a duplicate of the character appeared. When walking, only one character. Stopping made the duplicate reappear.

**Root Cause:** `LPCLayer.ANIM_INFO["idle"]` hardcoded `{"frames": 1}`, but LPC idle spritesheets are actually **2 frames × 4 directions** (128×256 px, not 64×256). With `layer_hframes = 1`, `AtlasTexture.region` spanned the full 128px width, rendering **both idle frames side-by-side** — creating an exact duplicate appearance.

**Why walk was fine:** Walk spritesheets are 576×256 (9 frames), so `fw = 576 / 9 = 64px` — each frame correctly isolated. Idle at `fw = 128 / 1 = 128px` showed the entire row.

**Verified dimensions (`find` + `file`):**
- 686 idle.png files: **128×256** (2 frames × 4 dirs)
- 789 walk.png files: **576×256** (9 frames × 4 dirs)  
- 656 run.png files: **512×256** (8 frames × 4 dirs)

**File changed:** `scripts/LPCLayer.gd` → `ANIM_INFO` constant
- Before: `"idle": {"frames": 1, "fps": 3.0}`
- After: `"idle": {"frames": 2, "fps": 3.0}`

## Key Learnings
- Godot 4.x `Color` lost `distance_to()` → use `is_equal_approx()` or manual component diff.
- GDScript indentation: tabs vs spaces will cause parse errors (enforces consistency).
- LPC spritesheet row order is asset-dependent — always verify with pixel analysis, don't assume documentation.
- `:=` type inference in GDScript can fail on string concatenation; use `=` for strings.
- Dynamic asset resolution (`resolve_asset_bases`) is always preferred over saved paths — it captures fg/bg layers and body-type-specific variants.
- **Always verify actual spritesheet frame counts against code assumptions.** `AtlasTexture.region` with wrong `hframes` will silently render multiple frames as one wide sprite, creating ghost/duplicate effects.
