# 2026-05-15 - Galaxy Demo Bug Fixes

## Hair Color Selection Fixed
- **Issue:** Could not select hair color in Character Creator or in-game
- **Root cause:** Hair was explicitly excluded from palette recoloring. Code treated hair as "pre-colored" (each color has its own PNG), but LPC hair spritesheets actually use the same source palette colors as body/head and need runtime recoloring.
- **Files changed:**
  - `scripts/CharacterCreator.gd` - `_get_available_colors()` now returns hair palette names; `_add_preview_layer()` recolors hair using hair palette; `_apply_layer_palette()` applies hair palette to hair layers
  - `scripts/Player.gd` - `_apply_palette_to_layer()` and `_apply_palette_to_textures()` now handle hair palette recoloring in-game
  - `scripts/OtherPlayer.gd` - Same hair color fixes for NPCs/other players

## NPC Invisibility Fixed
- **Issue:** NPCs were invisible - nametags moved but characters didn't render
- **Root cause:** `GameConfig.gd` had fatal GDScript parse errors preventing it from loading as an autoload. Godot 4.6 couldn't infer types for `:=` declarations from untyped Array/Dictionary lookups (e.g., `var bt := body_types[randi() % body_types.size()]` fails because `body_types` is untyped Array)
- **Fix:** Added explicit type annotations to all affected variables in `generate_random_lpc_character()` (14 variables total)
- **Verification:** Running `godot --headless --quit` now shows zero parse errors

## Missing NPC Heads Fixed
- **Issue:** NPCs had hair floating with no head underneath
- **Root cause:** `generate_random_lpc_character()` never added a head layer
- **Fix:** Added head layer selection right after body layer:
  - male/muscular/teen → "Human Male"
  - female/pregnant → "Human Female"
  - child → "Human Child" (but child was later removed, see below)

## Child Body Type Removed
- **Issue:** Child body type assets use completely different format from adult types
- **Root cause:** Adult assets have separate `walk.png`/`idle.png`/`run.png` files (standard LPC 4-direction layout). Child assets use single color-named PNGs with all animations packed into one tall spritesheet (e.g., `black.png` at 768×1344 = 12×21 frames — completely different slicing layout)
- **Fix:** Removed `child` from `BODY_TYPES` in both `CharacterCreator.gd` and `GameConfig.gd`
- **Remaining body types:** male, female, muscular, teen, pregnant

## Child Head Options Filtered
- **Issue:** Clicking male body type still showed child head options in Character Creator
- **Fix:** Added filter in `_refresh_section_content()` to skip any head option containing "child" in the name when body type is not child

## Clothing Idle Animations Generated
- **Issue:** Some clothing items (corset, blouse, tunic, etc.) appeared static during idle breathing while body moved
- **Root cause:** LPC library has ~16+ clothing items without idle animations. Items with only walk textures froze at frame 0 during idle, creating disconnected look.
- **Fix:** Generated synthetic idle spritesheets for 24+ clothing items by extracting walk frame 0 and creating a 1px-shifted breathing frame:
  - Frame 0 = walk frame 0 (standing pose)
  - Frame 1 = walk frame 0 shifted up 1px (subtle breathing)
  - Format: 128×256 (2 frames × 4 directions at 64×64)
- **Items fixed:** corset, blouse, blouse_longsleeve, robe, shirt, tunic, tunic_sara, collared, frock, iverness, santa, tabard, trench, trim/frock_buttons, trim/frock_collar, trim/frock_lace, trim/frock_lapel, trim/jacket_pockets, pants (child/female/pregnant/teen), skirts/child
- **Script:** `generate_idle.py` - can be re-run if more items need idle animations
- **Verification:** Idle frame 0 is identical to walk frame 0 (0% diff); idle frame 1 has 261/4096 pixel diffs (subtle breathing shift)

## LPC Asset Format Notes
- Walk: 576×256 (9×4 @ 64px) or 512×256 (8×4 @ 64px), some 1024×512 (8×4 @ 128px)
- Idle: 128×256 (2×4 @ 64px)
- Run: 512×256 (8×4 @ 64px)
- Child format is INCOMPATIBLE - single packed PNG with non-standard frame layout
- Colored items use `walk/color.png` format; flat items use `walk.png` format
