
## 2026-05-06 — Galaxy Demo Godot Port

### Key Technical Lessons

#### Godot 4 Coordinate System vs Phaser 3
- **Phaser 3**: `rectangle(center_x, center_y, width, height)` — center-based
- **Godot ColorRect**: `position` is top-left, `size` is width/height
- **Fix**: Created `_rect(cx, cy, w, h, color)` helper that converts center-based to top-left:
  ```gdscript
  func _rect(cx, cy, w, h, color):
      var r = ColorRect.new()
      r.position = Vector2(cx - w/2.0, cy - h/2.0)
      r.size = Vector2(w, h)
      r.color = color
      add_child(r)
  ```
- This caused the "grey background with grass in corner" bug because all room elements were placed at negative coordinates (centered around 0,0) while the viewport shows (0,0) to (960,600).

#### Godot 4 class_name vs Autoload Singletons
- `class_name` declarations conflict with autoload singletons of the same name
- **Fix**: Removed all `class_name` declarations from scripts that are also autoloaded
- Access autoloads directly by their registered name (e.g., `GameConfig.ROOMS`)
- Autoloads are configured in `project.godot` under `[autoload]` section

#### Godot 4 Node Lifecycle — @onready Timing
- **Critical**: `@onready var` is only initialized when the node ENTERS the scene tree
- Calling methods that access `@onready` vars BEFORE `add_child()` causes null reference errors
- **Pattern**: Always `add_child(node)` BEFORE calling `node.setup()` or any method that uses `@onready` vars
- **Wrong**: `bot.setup(); add_child(bot)`
- **Right**: `add_child(bot); bot.setup()`

#### Godot 4 SpriteFrames Default Animation
- `SpriteFrames.new()` already contains a "default" animation in Godot 4
- Calling `add_animation("default")` throws "SpriteFrames already has animation 'default'"
- **Fix**: Check `has_animation("default")` before adding, or just configure the existing one:
  ```gdscript
  if not frames.has_animation("default"):
      frames.add_animation("default")
  frames.set_animation_speed("default", fps)
  frames.set_animation_loop("default", loop)
  ```

#### Godot 4 Asset Import
- PNG files copied into `assets/` need `.import` files generated by Godot
- **Force import**: `/Users/Jakin/Downloads/Godot.app/Contents/MacOS/Godot --path <project> --headless --editor --quit`
- This scans all assets and generates import files in `.godot/imported/`

#### Node Method Name Conflicts
- `set_owner()` conflicts with Godot's native `Node.set_owner()` method
- **Fix**: Renamed to `set_pet_owner()` for the dog pet
- Similarly avoid naming methods that match built-in Node/Control methods

#### Persistent Account Storage
- Uses `user://accounts.cfg` (ConfigFile) for account data
- `user://` resolves to `~/Library/Application Support/Godot/app_userdata/<project_name>/` on macOS
- Account data stores: username, password, penguinColor (Dictionary with color), nameTagColor, createdAt
